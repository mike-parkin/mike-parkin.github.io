Problem Solving techniques

For my non tech-savvy friends

A simple problem tha got me stuck

I got stuck on a problem in a challenge called JS-Gradebook, this challenge was about taking names and grades and arranging them into a way that could contain both grades. The last challenge on this exercise was to write a piece of code that did all the work in one go, so you put the information into the code, and the code arranges it. Now I thought I had this one nailed, but errors were still being thrown, I was getting a little frustrated. I knew my logic was correct and that it was just a matter of me not using the correct syntax. Luckily I had my one on one with my facilitator at that time, so we got to look at it together and it turns out it was just a silly syntax error. What did I learn… that other people are helpful and syntax matters!

When I got all elegant solving a problem

My most elegant solving of a problem came in a challenge where we had to write code that would take a sentence and take the word buzz out of the sentence, this was the final of four challenges where we had to use similar methods to achieve similar results, so after being stuck on the last three challenges for ages. I think I’d finally got to grips with these new methods, I started by writing a small pseudo code then rolled writing the code with confidence, then Hey! It worked straight away! Except no…… It was only removing the word ‘buzz’ if it was lower case so ‘BuZZ’ & ‘buZz’ were still hanging around. I got talking to my little duck and figured out that I can convert all the letters to the same case and remove them that way.

How do I feel using various problem solving techniques?

Pseudocode

I feel good using pseudocode when beginning a new problem, however once I was deep into a problem then all pseudocode went out the window.

Trying Something

Not my favourite method, I would only try something as a last resort, so if I'm in that postion my confidence will have taken a hit or two.

Rubber Ducky Method

Love it, its funny and it makes your brain change tack. It gets you to think of the problem in a way you might never have used if you were just going over the problem in your head.

Reading the Error Messages

I don’t look at error messages as a big scary thing anymore, usually they can point you to the line and location of what’s throwing the error, so I definitely view them as more help than hindrance.

console.log()-ing

I got used to using this to check my variables were at their expected values at certain points in the code. I don’t think I could solve a problem with it but it helps me narrow down where the issue might be.

Googling

I found google extremely useful for finding out how the code methods and syntax work, but when using google to solve specific problems, well that came down to luck.

Asking Peers for Help

I haven’t reached out to my peers for help yet, but I’m confident that it’ll be the best method to use in the future

Asking Coaches for Help

My facilitator, John, was great helping me solve an issue (mentioned above). Definitely a great avenue to turn to for fixing issues

Reflection

This has been great to reassess my problem-solving skills outside the problem-solving environment. Using reflection, I’ve been able to get a better understanding of why my solutions have worked (especially if the answer came of the back of a google search).
